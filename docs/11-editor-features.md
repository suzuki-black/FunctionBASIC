# 11. エディタ機能仕様（IDE級・JetBrains風）

対応: エディタUXの最低限必須機能＋α。基本UI（行番号・×・タブ・保存）は [07-editor-ui.md](07-editor-ui.md)、
保存・エンコーディングは [08-file-save.md](08-file-save.md)、再生は [10-tech-stack.md](10-tech-stack.md) を参照。

---

## 11.1 方針

- **MSX-BASIC（構造化BASIC）専用エディタ**として、JetBrains等の高機能IDE相当の編集効率を目指す（操作系はJetBrains風）。
- ただし最優先は **「変換後 `.bas` が確実に生成・実行できること」**。編集支援は変換の妨げにならない範囲で提供する。
- 実装は **CodeMirror 6** を推奨（Tauri の WebView 上で動作、カスタム言語・分割・検索・折りたたみ・装飾を標準サポート、TSコアと親和。[10](10-tech-stack.md)）。
- 編集はUnicodeでもMSXへ渡す保存は **Shift-JIS（JIS X 0201＋0208）**（[08 §8.6](08-file-save.md#86-ファイルフォーマット詳細)）。

## 11.2 フォント・表示

- **等幅フォント必須**（桁位置・インデント・行番号の整列に不可欠）。既定は**システムの等幅フォント**
  （例: Windows=`Consolas`/`MS Gothic`、macOS=`Menlo`/`SF Mono`。日本語等幅にフォールバック）。
- **文字サイズはユーザが変更可能**（MSX愛好層・視認性に配慮）。
  - `Ctrl/Cmd + +` / `-` で拡大縮小、`Ctrl/Cmd + 0` で既定に戻す。設定 `fontSize`（[10 §10.9](10-tech-stack.md#109-設定settings)）に永続化。
- 現在行ハイライト・対応括弧ハイライトを既定オン（可読性）。

---

## 11.3 画面分割（上下2分割）

- エディタ領域を **上下2画面に分割**できる（水平スプリット）。
- 同一ファイルの別位置を同時表示（スクロール独立）、または「上=構造化ソース／下=別の関数」のような併読に使う。
- 変換後プレビュー（[07](07-editor-ui.md) のMSXタブ）とは別枠。分割は**構造化BASIC編集ペイン内**の上下分割を指す。
- 操作：分割トグル、ペイン間フォーカス切替、分割比のドラッグ調整（キーは設定可。§11.15）。
  JetBrainsの「Split and Move」相当（エディタタブを上下に分割）をイメージ。

```
┌ 構造化BASIC ───────────────────────┐
│ (上ペイン) 関数 FIND_ZERO 付近        │
├────────────────────────────────────┤
│ (下ペイン) 関数 MAIN 付近            │
└────────────────────────────────────┘
```

## 11.4 シンタックスハイライト

MSX-BASIC／構造化BASICの要素を色分けする。

| 種別 | 対象 |
|------|------|
| キーワード | `IF THEN ELSE END FOR TO STEP NEXT WHILE WEND GOTO GOSUB RETURN DIM DATA READ PRINT INPUT LET REF FUNCTION BREAK CONTINUE` 等 |
| 関数名 | `FUNCTION` 定義名・呼び出し名 |
| 文字列リテラル | `"..."`（ASCIIダブルクォート、[README §9](README.md#9-変換後コードの品質要件)） |
| 数値 | 整数・小数・`%`/`$` サフィックス付き変数 |
| コメント | `'` / `REM` 行（**大文字化・変換の対象外**） |
| 演算子・区切り | `+ - * / = < > <= >= <> : ; , ( )` |

- ハイライト用の語彙は Lexer（[03](03-lexer-parser.md)）のトークン種別と共有し、二重定義を避ける。
- **目的**：可読性向上とミスの早期発見。

## 11.5 エディタ行番号

- 左ガターに **エディタ物理行番号** を常時表示（MSX行番号とは独立）。詳細は [07 §7.2](07-editor-ui.md#72-エディタ行番号の常時表示仕様5-1)。
- 用途：文法エラー位置の提示、変換テーブルのキー、逆変換の行対応。

---

## 11.6 Undo / Redo

- 編集操作の取り消し・やり直し（`Ctrl/Cmd + Z` / `Ctrl/Cmd + Shift + Z` または `Ctrl + Y`）。
- 内部はコマンド履歴（変更トランザクション列）を保持。CodeMirror の履歴機能を利用。
- 履歴はファイル単位。保存しても履歴は保持（保存=履歴クリアにしない）。

---

## 11.7 検索・置換（正規表現対応）

- インクリメンタル検索、全置換、選択範囲内置換。
- **正規表現を最低限サポート**（`\d` `\w` `.*` 後方参照 `$1` 等）。
- 大文字小文字の区別トグル（既定は区別しない＝自動大文字化言語のため）。
- 用途：変数名の一括変更、構造修正。**置換結果が Shift-JIS 表現可能か**は保存時に検査（§11.13）。

---

## 11.8 インデント制御

- `Tab` / `Shift+Tab` でインデントの増減（選択行まとめて可）。
- **ブロック構造に応じた自動インデント**：`IF…END IF` / `FOR…NEXT` / `WHILE…WEND` / `FUNCTION…END FUNCTION`
  に入るとインデント+1段、閉じで-1段。ネスト許可（[01](01-language-spec.md#14-ブロック構造とネスト仕様1-4ネスト許可へ改訂)）に対応し**多段ネストを正しく字下げ**する。
- インデント幅は設定可能（既定スペース2 or 4）。タブ→スペース展開を既定。
- 自動フォーマッタ（§11.14 A）と整合（同じインデント規則）。

---

## 11.9 入力補完

- **括弧の自動補完**：`(` 入力で `)` を補完、`"` で閉じ `"` を補完。
- **文字列の閉じ忘れ補完**：行末で未閉鎖の `"` を検出し補完（Lexerの `E_UNTERMINATED_STRING` を未然に防ぐ）。
- **予約語補完（任意）**：先頭数文字から候補表示（`FOR` `FUNCTION` `PRINT` 等）。Tab/Enterで確定。
- 補完語彙はシンタックスハイライト・Lexerと共有。

## <a name="1110-ナビゲーション"></a>11.10 ナビゲーション（JetBrains 風）

ナビゲーションは Parser が生成する **シンボルインデックス**（[04 §4.8](04-data-model.md)）を用いる。
AST から関数定義・呼び出し箇所・変数定義（初期化）・参照を抽出して索引化する。
操作系・用語は **JetBrains IDE（IntelliJ/Rider 等）流**に揃える（vim流のマーク／ジャンプリストは採用しない）。

| 機能 | 既定キー（Win/Linux ／ macOS） | 動作 |
|------|-------------------------------|------|
| **定義へ移動**（Go to Declaration） | `Ctrl + B` ／ `Cmd + B`（`Ctrl/Cmd + クリック`も可） | カーソル下の識別子の定義（関数定義／変数初期化）へ飛ぶ |
| **使用箇所の検索**（Find Usages） | `Alt + F7` | 関数の呼び出し元・変数の参照箇所を一覧表示して選択ジャンプ |
| **戻る／進む**（Navigate Back/Forward） | `Ctrl + Alt + ←/→` ／ `Cmd + Alt + ←/→` | カーソル位置の移動履歴を前後（JetBrainsのBack/Forward） |
| **最近の位置**（Recent Locations） | `Ctrl + Shift + E` ／ `Cmd + Shift + E` | 最近編集/閲覧した箇所から選択ジャンプ |
| **行へ移動**（Go to Line） | `Ctrl + G` ／ `Cmd + L` | **エディタ行番号**（MSX行番号ではない）を入力して移動 |
| **ファイル構造**（Structure popup） | `Ctrl + F12` ／ `Cmd + F12` | 関数一覧をポップアップ表示してジャンプ |
| **ブックマーク切替** | `F11` | 行にブックマークを付与/解除（JetBrainsのBookmark） |
| **ニーモニック・ブックマーク** | `Ctrl + F11` ／ `Cmd + F11` | 数字/文字付きブックマークを付与し、その番号キーで即ジャンプ |
| **ブックマーク一覧** | `Shift + F11` | ブックマーク一覧から選択ジャンプ（**永続化**：プロジェクト設定に保存） |

- **関数：呼び元↔呼び先** — 定義上で `Ctrl/Cmd + B`＝定義へ、`Alt + F7`＝呼び出し元一覧。シンボルインデックスの `defLine`/`callSites` を辿る。
- **変数の「初期化箇所」へ** — 変数上で `Ctrl/Cmd + B`。当該スコープ（関数 or トップレベル）で**最初に代入（初期化）している行**（`LET`／`DIM`／`REF`仮引数）へ飛ぶ。
- **行移動は エディタ行番号基準**（変換後のMSX行番号とは無関係。混同しないようダイアログで「エディタ行番号」と明示）。
- **ブックマークはプロジェクト設定に永続化**（再起動後も保持）。vim流の `marks`/`jumplist` ではなく、JetBrainsのBookmarks＋Back/Forward＋Recent Locationsで構成する。

## 11.11 エラーチェック（保存時診断）

- 保存時に文法チェック＋変換時チェックを実行し、エラー行に印を付ける（[07 §7.3](07-editor-ui.md#73-エラー表示仕様5-2)）。
  - 行番号横に **×**、マウスオーバーでエラー内容（行:列・種別・メッセージ）。
  - **最初のエラー行へ自動スクロール**。
- 対象エラー：文法（[03](03-lexer-parser.md)）＋変換時（`E_VAR_NAMES_EXHAUSTED` / `E_LINE_TOO_LONG` / `E_RECURSION_UNSUPPORTED` / `E_NON_SJIS`）。
- 入力中はデバウンスで増分チェック（プレビュー更新）、保存時に確定（[07 §7.6](07-editor-ui.md)）。

## 11.12 変換プレビュー表示

- エディタとは別枠（**MSX-BASIC変換後タブ**、読み取り専用）に変換結果を表示（[07 §7.4](07-editor-ui.md#変換後モード)）。
  - 行番号付き／コピー可能／エラー時は更新せずエラー表示。
- WebMSX等へ貼り付けて即実行できる形式（Shift-JIS・CRLF・行番号付き、[08](08-file-save.md)・[10](10-tech-stack.md)）。

## 11.13 エンコーディング・改行

> **【仕様変更】** 当初の項目8「UTF-8固定/LF固定」は撤回。**MSXに渡す保存は Shift-JIS（JIS X 0201＋0208）** とする。
> MSX実機の符号化（漢字=JIS X 0208 / 半角カナ=JIS X 0201、`CALL SJIS`）に最も忠実なため。

- **エディタ内部はUnicode**で編集し、**保存時に Shift-JIS へ変換／読込時に Shift-JIS から復元**する。
- ファイル別エンコーディング（詳細は [08 §8.6](08-file-save.md#86-ファイルフォーマット詳細)）：
  - `.msxb`（構造化ソース）：**Shift-JIS / LF**
  - `.bas`（MSX出力）：**Shift-JIS / CRLF**（MSX実機・WebMSX・エミュレータ互換を最優先）
  - `.map.json`（変換テーブル）：**UTF-8 / LF**（JSON標準）
- **Shift-JISで表現できない文字**（CP932固有拡張・絵文字・MSX非対応記号等）を含む場合は保存・変換が失敗するため、
  **保存前に検査して `E_NON_SJIS` で通知**（該当行を×表示）。これにより「変換時の失敗」を未然に防ぐ。
- **CP932は既定では採用しない**：MSXに無いベンダ拡張を含み、`0x5C`の扱い（JIS X 0201では¥）も異なるため、
  素のShift-JISに制限する方が忠実・安全。ただし設定 `encoding=cp932` で任意選択は可能（[10 §10.9](10-tech-stack.md#109-設定settings)）。

## 11.14 追加で強く推奨（最低限＋α）

### A. 自動フォーマッタ
- 保存時／フォーマット実行時にインデント・スペースを整形（§11.8 のインデント規則、Formatter [02](02-architecture.md)）。
- **キーワード・変数名・関数名の大文字化もこのタイミングで実施**（[01 §1.6](01-language-spec.md#16-自動大文字化仕様1-6)）。
  - **タイプ中は大文字化しない**（カーソル/IME変換のちらつき防止）。打った通りに表示し、保存/フォーマットで一括変換。
  - **文字列リテラル・コメントの大小は変更しない** — あえて小文字にした箇所はそのまま保持する。

### B. 折りたたみ（コードフォールディング）
- `FUNCTION…END FUNCTION` / `IF…END IF` / `FOR…NEXT` / `WHILE…WEND` をブロック単位で折りたたみ。
- ネスト許可に対応し、入れ子ブロックも個別に折りたためる。

### C. スニペット
- `FOR…NEXT`・`IF…END IF`・`FUNCTION…END FUNCTION` 等のテンプレート展開（プレースホルダ・Tab送り）。

### D. 逆変換差分表示
- 変換前（構造化）と、変換後を逆変換した結果の差分を可視化（[06](06-reverse-transformer.md)）。
- 往復一致しない箇所（最適化で戻らない式・コメント位置ズレ等、[06 §6.10](06-reverse-transformer.md#610-逆変換の限界と注意)）の確認に使う。

## 11.16 行対応ハイライト（構造化 ⇔ 変換後）

**目的**：構造化BASICの各行が、変換後MSX-BASICのどの行になるかを一目で示す。変換の仕組みを学ぶ／実装者が対応を確認する用途。

- **動作条件**：**分割表示（構造化ペインとMSX-BASICペインの両方が同時に見えている）時のみ**。片方だけ表示中はキャレット移動で何も起きない（勝手にタブを開いたり移動・ハイライトしない）。
- **双方向**：
  - 構造化側でキャレットを行に移動 → 対応するMSX行を緑ハイライトし、その行が見える位置までMSXペインをスクロール。ヘッダに `構造化 N行 → MSX M行` と明示。
  - MSX行をクリック → 由来の構造化行へキャレットを移動（構造化側にネイティブ現在行ハイライト）。
- **出力を生まない行**：`GLOBAL` 宣言・コメント・空行・未呼び出し関数の本体など、変換後に対応行が無いソース行では「変換後に出力はありません」とヘッダに表示（無反応で壊れていると誤解させない）。
- **1行に畳まれた `IF`**（`IF cond THEN a: b`）：畳まれたMSX行は、`IF` と内側の全文（`THEN`/`ELSE` 側）すべてのソース行を由来に持つ。どの内側行をクリックしても対応が出る。

**土台となるデータ**：トランスパイラは各 `MsxLine` に `src?: number[]`（その行の由来＝構造化ソース行番号）を付与する（[04 データモデル](04-data-model.md)／[05 トランスフォーマ](05-transformer.md)）。エディタはこの `src` から「構造化行→MSX行インデックス」の対応表を作り、上記ハイライトに用いる。回帰防止に `test/provenance.test.ts`（由来の健全性・FUNCTION宣言→見出し行・畳みIF・行範囲内）を持つ。

**行高の較正（デスクトップ WKWebView 対策）**：構造化ペインは `<textarea>`（キャレット）に色付き `<div>` オーバーレイ（#highlight）とガターを重ねて実装している。WKWebView は `<textarea>` をフォント固有の自然行高で描画する一方、`<div>` の `line-height` を整数pxに丸めるため、両者が約 0.05px/行ずれ、下の行ほどキャレットが色付きテキスト・行番号からズレる（900行付近で約2行）。対策として起動時／読込時／フォント変更時／リサイズ時に `<textarea>` の実描画行高を実測し、**自然行高より確実に大きい整数へ `--line` を引き上げる**（`calibrateLineHeight()`）。これで `<div>` も `<textarea>` も同じ整数を厳密に honor して一致し、丸めの影響を受けない。ブラウザ（Chromium）では実測行高が整数のため no-op。
