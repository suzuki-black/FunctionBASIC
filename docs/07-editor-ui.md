## 7.2 エディタ行番号の常時表示（仕様5-1）

- 左ガター(gutter)に **エディタ物理行番号** を常時表示する。
- これは **MSX行番号（100,110,…）とは無関係** の、編集上の行番号である。
- 行の折り返しがあっても論理行単位で1番号。

```
┌────┬──────────────────────────┐
│  1 │ FUNCTION ADD(A, B)        │
│  2 │     RETURN A + B          │
│  3 │ END FUNCTION              │
└────┴──────────────────────────┘
```

---

## 7.3 エラー表示（仕様5-2）

- 文法エラーがある行の **右端に `×`** を表示する。
- `×` に **マウスオーバー** すると、その行のエラー内容を **ツールチップ** で表示する。
- 同一行に複数エラーがある場合はツールチップ内に改行で列挙する。

```
│  2 │     RETRN A + B        ×│   ← マウスオーバーで↓
                              ┌──────────────────────────────┐
                              │ 行2:10 E_UNKNOWN_KEYWORD       │
                              │ 「RETRN」は不明な語です。       │
                              │ RETURN の誤りではありませんか？ │
                              └──────────────────────────────┘
```

実装:
- 文法チェック結果 `EditorErrorModel[]`（[04](04-data-model.md#45-エラーモデル)）を行番号で索引。
- ガター右端のマーカー領域に `×` 要素を配置、`title`/カスタムツールチップで `tooltip` を表示。
- エラーは入力デバウンス（例 400ms）で増分更新し、保存時にも再計算する。

## 7.4 タブ（表示モード）（仕様5-5, 5-6, 5-7）

### 2つのタブ

| タブ | 内容 | 編集 |
|------|------|------|
| 構造化BASICモード | 入力・編集する本体 | 編集可 |
| MSX-BASIC変換後モード | 変換結果の表示 | **読み取り専用**（仕様5-6） |

### <a name="変換後モード"></a>変換後モードの挙動（仕様5-6, 5-7）

- **文法チェックと変換の両方が成功した場合のみ** MSX-BASICコードを表示する（仕様5-6）。
- エラーがある場合は変換後タブに「文法エラーのため変換できません。構造化BASICタブを確認してください」と表示。
  - 文法は正しくても変換時に `E_VAR_NAMES_EXHAUSTED`（2文字MSX名を使い切り）や
    `E_LINE_TOO_LONG`（1行255バイト超過、[05 §5.12](05-transformer.md#512-1行255バイト制限の検査と自動分割)）、
    `E_RECURSION_UNSUPPORTED`（再帰）等で変換できない場合も同様に、変換後タブは更新せずエラーを表示する（[08](08-file-save.md)）。
- 表示コードは **コピー可能**（コピーボタン＋範囲選択可）。
  - WebMSX 等のブラウザエミュレータに貼り付けて即実行できる形式（CRLF改行・行番号付き）（仕様5-7）。
- 「コピー」ボタンでクリップボードへ全文コピー。

```
┌ MSX-BASIC変換後（読み取り専用）─────────────[📋コピー]┐
│ 100 ' === MAIN ===                                   │
│ 110 AA=1 : AB=2 : GOSUB 1000                         │
│ 120 X=AR                                             │
│ …                                                    │
└──────────────────────────────────────────────────────┘
```

## 7.5 保存ボタンの動作（仕様5-3, 5-4）

保存ボタン押下時:

```
保存押下
  │
  ▼ 文法チェック実行
  ├─ エラーあり ─▶ 変換前ファイルのみ保存（変換不可）
  │                 ・エラー行に×表示
  │                 ・ステータス「誤りがあります。確認してください」
  │
  └─ エラーなし ─▶ 文法チェック → 変換 → 変換前保存 → 変換テーブル保存 → 変換後保存
                    ・変換後タブにMSXコード反映
                    ・ステータス「保存しました」
```

- エラーありでも変換前は保存される（仕様1-8/6-2）→ 作業内容は失われない。
- メッセージ仕様（仕様5-4）:
  - 成功: **「保存しました」**
  - エラー: **「誤りがあります。確認してください」**

詳細フローは [08-file-save.md](08-file-save.md)。

## 7.6 操作と状態遷移

```
            編集
   [編集中] ───────▶ [編集中(dirty)]
       │  入力停止(400ms)
       ▼
   文法チェック ── errors>0 ──▶ ×表示更新, 変換後タブ=不可
       │ errors=0
       ▼
   変換後タブ表示可（プレビュー更新）
       │ 保存押下
       ▼
   保存（§7.5）→ [保存済]
```

- タブ切替時、変換後タブを開いた瞬間に最新の文法チェック結果で表示可否を判定。
- 未保存(dirty)で閉じようとした場合は確認ダイアログ。

## 7.7 キーボード／アクセシビリティ

| 操作 | キー |
|------|------|
| 保存 | Ctrl/Cmd + S |
| タブ切替 | Ctrl/Cmd + Tab |
| 変換後コピー | 変換後タブで Ctrl/Cmd + C（全選択時） |

- `×`マーカーは `aria-label` にエラー文言を設定し、スクリーンリーダ対応。
- ツールチップはフォーカス時にも表示（キーボード操作対応）。

---

## 7.8 エラー表示の配色・体裁（推奨）

| 要素 | 体裁 |
|------|------|
| エラー行 `×` | 赤系・右端固定 |
| 警告（severity=warning） | 黄系・△ |
| ツールチップ | `行:列 種別` ＋ メッセージ（日本語） |
| 変換後タブ（不可時） | 中央にグレーのプレースホルダ文言 |

## 7.9 UI コンポーネント分割（実装指針）

| コンポーネント | 責務 |
|----------------|------|
| `Editor` | テキスト入力・カーソル・選択（CodeMirror 6 ベース。シンタックスハイライト・補完・折りたたみ） |
| `Gutter` | エディタ行番号表示（仕様5-1） |
| `ErrorMark` | 行右端の×・ツールチップ（仕様5-2） |
| `Tabs` | 構造化/変換後 切替（仕様5-5） |
| `MsxView` | 変換後の読み取り専用表示＋コピー（仕様5-6,5-7） |
| `SplitView` | 構造化編集ペインの上下2分割（[11 §11.3](11-editor-features.md#113-画面分割上下2分割)） |
| `Navigator` | 定義へ移動／使用箇所検索／変数初期化箇所／行へ移動／ブックマーク（JetBrains風、[11 §11.10](11-editor-features.md#1110-ナビゲーション)・[04 §4.8](04-data-model.md#48-シンボルインデックスエディタナビゲーション用)） |
| `StatusBar` | 保存結果メッセージ（仕様5-4）・カーソル位置・エンコーディング表示（Shift-JIS） |
| `Toolbar` | 保存・開く・ファイル名・フォントサイズ |

> **本章は基本UI（仕様5）を定義する。** シンタックスハイライト・Undo/Redo・検索置換・インデント・補完・
> 画面分割・ナビゲーション・フォント（等幅・サイズ可変）・エンコーディング等の編集機能の詳細は
> [11-editor-features.md](11-editor-features.md) に集約する。

状態管理は [02-architecture.md](02-architecture.md#26-状態管理editorui) を参照。
